FFAB Overview

Current Version: 0.1.1

FFAB is a GUI wrapper for FFmpeg that makes designing complex audio filter chains easily accessible to non-command-line users.

The app creates, saves, shares, and modifies complex filter chain "presets" – without embedding FFmpeg code - FFAB locates and calls the user's installed FFmpeg binary.

App Architecture

Visual Filter Chain Builder

Drag-and-drop filter ordering with vertical "effects rack" design

Sidechain Input & Aux Output

Hardcoded & custom flexible routing of parallel audio streams

Real-time Preview

Generate audio preview with waveform visualization; scrubbing playhead & payback region select

Batch Processing

Process multiple File Lists with 7 different pairing algorithms

Video Passthrough

Input video files, add audio-only effects, then embed new audio with untouched video

Preset System

Save/load complete processing states including filter chains filter params, settings, and file lists (optional)

FFmpeg Audio Filters

Almost all standard filters, including analysis & advanced utility filters

Command Line Integration

Audio Previews

Hear an instant preview of yor FFmpeg filter chain,  plus a helpful waveform imag. Click [Generate Preview] in either the main app window or from within the larger pop-out Region Preview window.

The Generate Preview widget

The pop-out Region Preview window has tools tohighlight a region of audio – more representative of the entire file than say, the intro – enabling more meaningful auditioning from the File List. This is very helpful when auditioning longer duration files (podcast dialong, archive recordings etc).

The Region Preview pop-out
  • Waveform visualization - Visual representation of audio
  • Click-to-seek - Click waveform to jump to position
  • Drag-to-scrub - Drag playhead for precise positioning
  • Region selection - ALT+drag to select region for focused playback
  • Region Preview Window - Expanded view for detailed scrubbing
  • View FFmpeg command - Syntax-highlighted command display
  • Time display - Current position and total duration

Batch Processing Modes

#1 Sequential

Apply chain to every file: N → N
Use Cases:
Batch Algorithm #1: Sequential
  • Convert a folder of 24/96k AIFF -> FLAC & MP3 & 16/44k WAV all in one pass
  • Normalize dialog on 100+ raw video files without transcoding video
  • Apply EQ, Compression, DC Offset, noise reduction on 1000+ audio files

#2 Iterate

Re-process each file R times with gain reduction: N × R
Batch Algorithm #2: Iterate / Photocopy
Use Cases:
  • Photocopy of a photocopy of a photocopy...
  • Sound design, digital degredation analysis, feedback loops
  • Get creative!

#3 Zip

1:1 pairing of main + aux files
min(M,A)
Batch Algorithm #3: Zip / Pairing
Use Cases:
  • Input two folders: 001-mono-L.wav + 001-mono-R.wav -> 001-stereo.wav
  • Insert custom audio soundbites on a per-file basis
  • Complex 5.1 or multichannel audio tasks

#4 Broadcast Fixed

Single aux applied to all main files: N
Batch Algorithm #4: Broadcast Fixed
Use Cases:
  • Watermark a folder of audio files with one ident
  • Demo one IR reverb against many audio files
  • Duck one background music file below numerous dialog files

#5 Broadcast Random

Random aux per main file: N
Batch Algorithm #5: Broadcast Random
Use Cases:
  • Apply random background foley to
  • Sound design, inspiration generator
  • Embrace chaos...

#6 Cartesian

Every main × every aux combination: N × M
Batch Algorithm #6: Cartesian / one to many
Use Cases:
  • Test 100+ IR Impulse Response files against one test signal
  • Generative music: Create random generative algos
  • Sound design: Mash up your sample library

#7 Cartesian Triple

Three-way product: N × M × P
Batch Algorithm #7: Cartesian / many to many (to many!)
... fill up disk storage in no time
Use Cases:
  • Combine a few source files -> new unique library
  • Inspiration, generative audio, sound design
  • Combine drums hits with timing & trim to generate new loops
  • We do it because we can, not because we know why

Command Building

All FilterChain command building happens with these key methods:

buildCompleteCommand() <- multiple overloads:

2-param: (inputFile, outputFile)
3-param: (inputFile, outputFile, mutedPositions)
3-param: (inputFile, sidechainFiles, outputFile)
4-param: (inputFile, sidechainFiles, outputFile, mutedPositions)

buildPreviewCommand() <- Generates preview with aux outputs discarded to null muxer

buildFilterFlags() <- Builds the -filter_complex "..." section

buildAuxOutputPath() <- Determines output paths for aux files

Who Calls FilterChain?

BatchProcessor calls buildCompleteCommand()
MainWindow calls buildCompleteCommand() for View Command
PreviewGenerator calls buildPreviewCommand() for audio preview
